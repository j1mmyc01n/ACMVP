# PWA Testing Guide

This document provides instructions for testing the Progressive Web App (PWA) functionality of Acute Connect.

## What Was Added

The application now includes full PWA support:

1. **Web App Manifest** (`manifest.webmanifest`)
   - App name, description, and branding
   - Icons in multiple sizes (72px to 512px)
   - Display mode: standalone
   - Theme color: #4F46E5 (indigo)

2. **Service Worker** (auto-generated by Vite PWA plugin)
   - Offline support with precaching
   - Runtime caching for:
     - Google Fonts
     - Images
     - API calls
   - Automatic updates

3. **App Icons**
   - 8 different sizes (72x72 to 512x512)
   - Suitable for all devices
   - Maskable for adaptive icons

## Testing Instructions

### 1. Development Mode

```bash
npm run dev
```

The PWA features are enabled in development mode. Open the browser DevTools and check:
- **Application** tab → **Manifest**
- **Application** tab → **Service Workers**

### 2. Production Build

```bash
npm run build
npm run preview
```

Then visit `http://localhost:4173` (or the port shown).

### 3. Testing Installation

#### Desktop (Chrome/Edge)
1. Visit the site
2. Look for the install icon in the address bar (⊕ or install icon)
3. Click it to install
4. Or use the app's install button if the prompt appears

#### Mobile (Android)
1. Visit the site in Chrome
2. Tap the menu (⋮)
3. Select "Add to Home screen"
4. The app will install and appear on your home screen

#### Mobile (iOS)
1. Visit the site in Safari
2. Tap the Share button
3. Select "Add to Home Screen"
4. Note: iOS has limited PWA support compared to Android

### 4. Testing Offline Functionality

1. Install the app
2. Open DevTools → Network tab
3. Enable "Offline" mode
4. Refresh the page
5. The app should still load (cached content)

### 5. Verify PWA Features

Check these in Chrome DevTools → Lighthouse:

1. Run a Lighthouse audit
2. Select "Progressive Web App" category
3. Review the PWA score and suggestions

## Deployment

When deploying to production:

1. Ensure HTTPS is enabled (required for service workers)
2. The build files in `dist/` include:
   - `manifest.webmanifest`
   - `sw.js` (service worker)
   - `workbox-*.js` (service worker library)
   - `registerSW.js` (registration script)
   - All icon files

3. No additional server configuration needed - Vite handles everything

## Troubleshooting

### Service Worker Not Registering
- Check browser console for errors
- Ensure HTTPS is enabled (localhost is exempt)
- Clear browser cache and reload

### Install Prompt Not Showing
- The prompt only appears once per origin
- Clear site data in DevTools → Application → Clear storage
- Try a different browser or incognito mode

### Offline Mode Not Working
- Check DevTools → Application → Service Workers
- Verify files are cached in Cache Storage
- Ensure service worker is activated

## Customization

To customize the PWA:

1. **Change App Icon**: Replace `public/icon.svg` and regenerate with `node generate-icons.js`
2. **Update Manifest**: Edit `vite.config.js` → `VitePWA.manifest`
3. **Modify Caching**: Edit `vite.config.js` → `VitePWA.workbox`
