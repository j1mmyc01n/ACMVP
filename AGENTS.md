# Agent Operating Rules

This repository is a real SaaS product. Act like a senior full-stack engineer.

Do not produce fake implementation summaries. If you did not change a file, say so.

Prefer working code over explanation.

Before editing, inspect relevant files. After editing, run build checks when available.

Keep changes scoped, reversible, and production-safe.

Use existing:
- routing patterns
- auth patterns
- API patterns
- database client
- styling system
- component structure

Do not create parallel duplicate systems.

If a task is too broad, break it into the first safe implementation step and complete that step.
